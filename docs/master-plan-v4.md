# Master Plan v4: 大賢者を育てる = 稼ぐ

**日付**: 2026-02-08
**コンセプト**: 大賢者を育てる過程そのものがコンテンツであり、収益源

---

## 核心の考え方

```
大賢者を育てる → その過程を発信する → 読者が集まる → テンプレートが売れる
      ↑                                                    ↓
      ←←←←←←←← 売上で投資を回収 ←←←←←←←←←←←←←←←←←←←←←←←←←←
```

**大賢者の開発と収益活動は同じ行為。分離しない。**

### 最強のストーリー

```
大賢者が新機能を獲得する
  → その機能を使って実際に商品を作る（動作検証）
  → 作った商品がBOOTHで売れる
  → 「大賢者が作って売れた」を発信する
  → 発信を見た人が大賢者テンプレートを買う
```

**「大賢者が作ったもので実際に売れた」が最強の社会的証明。**
各Phaseで必ず「大賢者で作って売る」サブタスクを入れる。

## 4. フェーズ計画

### Phase 0: 土台整備（今週中）— 大賢者が動ける状態にする

**目的**: 公式ベストプラクティスに準拠し、大賢者が精度を落とさず動ける基盤を作る

| # | タスク | 所要時間 | 何が変わるか |
|---|--------|---------|------------|
| 0-1 | Skill 3層構造リファクタリング | 2h | 精度向上、コンテキストコスト削減 |
| 0-2 | サブスキルにfrontmatter追加 | 30min | Claudeの自動読み込み精度向上 |
| 0-3 | 不要ファイル・重複定義の削除 | 1h | プロジェクト整理 |
| 0-4 | registry.yaml を実態に合わせて更新 | 30min | 大賢者が正しい現状を認識 |
| **0-5** | **動作検証: /great-sage を呼んでプロジェクト状況レポート生成** | **30min** | **大賢者が実際に動くことを確認** |

**Skill 3層構造の設計**:

```
Layer 1: コア SKILL.md（50行以下、常時読み込み）
  → 大賢者の基本振る舞い、サブスキルへの参照のみ

Layer 2: アクションスキル（各100行以下、/コマンドで呼出）
  → /orchestrate: タスク生成・分解（orchestrator.mdから圧縮）
  → /content: コンテンツ管理（content-manager.mdから圧縮）
  → /mcp-scan: MCP探索（mcp-discovery.mdから圧縮）
  → /marketing: SNS分析・投稿提案（新規作成）
  → 全て disable-model-invocation: true

Layer 3: リファレンス（docs/配下、必要時のみ@importで参照）
  → docs/great-sage/orchestrator-guide.md（詳細プロトコル）
  → docs/great-sage/content-manager-guide.md（詳細プロトコル）
  → docs/great-sage/mcp-discovery-guide.md（評価基準・ソースリスト）
```

**精度を守るルール**:
- Skill は全て200行以下
- アクションスキルは `disable-model-invocation: true`
- 機能追加時は同量を削るか分離する
- CLAUDE.md は50行以下を維持

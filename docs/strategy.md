## 1. プロジェクト概要

### 1.1 オーナー情報

- **名前：** 藤原由大
- **状況：** 会社員として働きながら、個人で収益を立てたい
- **使用環境：** MacBook（2026年購入）+ Claude Max Plan（$200/月）
- **主要ツール：** Claude Code（自律的なプロダクト開発の中核）

**「Build in Public × ストック型収益」**

Claude Codeの圧倒的な開発速度を武器に、個人開発プロダクトを高速で量産・検証しながら、その過程を全て公開（Build in Public）してインフルエンサーポジションを獲得する。

## 2. 戦略の根幹：なぜこの組み合わせが最適か

### 2.1 Claude Codeが生む非対称優位性

| 従来の個人開発者 | Claude Code活用者 |
|---|---|
| MVP作成に2-4週間 | **2-3日でMVP完成** |
| 1プロダクトに集中 | **並行して3-5プロダクト検証可能** |
| コード書く時間でコンテンツ作れない | **開発とコンテンツ発信を同時進行** |
| 技術スタック制約あり | **フルスタック何でも対応** |
| バグ修正に時間を取られる | **高速デバッグ＆リファクタ** |

### 2.2 コンテンツ駆動型プロダクトビジネスのフライホイール

```
発信（Twitter/Zenn）
  → 人が集まる（フォロワー増加）
    → プロダクトが売れる（収益）
      → 実績ができる（信頼）
        → 発信のネタになる（コンテンツ）
          → もっと人が集まる ←──（ループ）
```

**影響力だけでは稼げない。プロダクトだけでは知られない。両方を回して初めてフライホイールが回る。**

### 2.3 なぜ「個人開発 × 技術発信」のハイブリッドか

各チャネルの単体評価：

| チャネル | ストック性 | 影響力構築 | Claude Code活用度 | 初期コスト |
|---|---|---|---|---|
| 個人開発（SaaS/ツール） | ◎ | △ | ◎ | 低 |
| Twitter/X | △ | ◎ | ○ | 最低 |
| Zenn / 技術ブログ | ◎ | ○ | ◎ | 低 |
| note | ○ | ○ | ◎ | 低 |
| YouTube | ◎ | ◎ | △ | 高 |

**結論：** Twitter/X + 個人開発 + Zenn の3本柱がROI最大。YouTubeはフォロワー1,000人超えてから参入。

## 3. チャネル戦略の詳細

### 3.1 チャネル優先順位

## 6. 技術スタック＆ツール

### 6.1 開発スタック

| レイヤー | 技術 | 理由 |
|---|---|---|
| Frontend | Next.js (App Router) + Tailwind CSS + shadcn/ui | 最速でモダンなUI構築。Vercelとの親和性◎ |
| Backend | Supabase (Auth, DB, Edge Functions) | BaaS最有力。無料枠が大きく、個人開発に最適 |
| Hosting | Vercel | Next.jsとの相性最高。無料枠で十分スタート可能 |
| Payment | Stripe | 国際標準。サブスクリプション管理が容易 |
| Analytics | Plausible or Umami (self-hosted) | プライバシーファースト。軽量 |
| Domain | Cloudflare | DNS管理 + CDN + Pages も使える |

### 6.2 マーケティングツール

| 用途 | ツール |
|---|---|
| SNS管理 | Twitter/X ネイティブ or Buffer（無料枠） |
| 技術記事 | Zenn |
| 有料コンテンツ | Zenn Book / note |
| テンプレート販売 | Gumroad / BOOTH |
| メール配信 | Resend（将来的） |
| 分析 | Twitter Analytics + Zenn ダッシュボード |

### 6.3 開発ワークフロー

```
1. Claude Code でMVP構築（2-3日）
2. Vercel にプレビューデプロイ
3. Twitter/X で告知・フィードバック収集
4. Claude Code でイテレーション
5. main ブランチにマージ → 本番デプロイ
6. Zenn に技術記事（開発過程の共有）
```

## 10. リスクと対策

| リスク | 影響度 | 発生確率 | 対策 |
|---|---|---|---|
| SaaSが全く売れない | 高 | 中 | テンプレート＋コンテンツで最低ラインの収益確保。3つ同時検証で確率を上げる |
| 本業が忙しくなる | 中 | 高 | Claude Codeの自律性を最大活用。週末集中型にシフト可能な体制を構築 |
| 競合の参入 | 中 | 中 | 先行者利益＋コミュニティ構築で差別化。「人」で差がつく領域（Build in Public）に注力 |
| SNSが伸びない | 中 | 中 | 投稿の質・量を見直し。バズった投稿のパターンを分析してPDCA |
| 技術的な壁 | 低 | 低 | Claude Codeがほぼ解決。解決できない場合はスコープ縮小 |
| バーンアウト | 中 | 中 | 自動化を徹底。「楽しい」と思えるプロダクトだけに注力。無理なペースは避ける |
| Claude API のコスト増 | 低 | 低 | SaaSでAPI利用する場合はユーザー課金でカバー。無料枠内に収まる設計を心がける |

## 11. 成功のための鉄則

### 鉄則1：「完璧を目指すな、出荷しろ」
Claude Codeがあれば80%のクオリティのプロダクトを数日で作れる。100%を目指して1ヶ月かけるより、80%を5つ出して市場に聞く方が圧倒的に勝率が高い。

### 鉄則2：「数字を公開しろ」
日本の個人開発コミュニティでは、売上やユーザー数を透明に公開する開発者が信頼とフォロワーを獲得する。「Claude Codeで投資回収チャレンジ」として過程を発信すること自体がマーケティングになる。

### 鉄則3：「Claude Codeに任せられることは全部任せろ」
人間の仕事は「何を作るか決める」「市場と対話する」「方向性を判断する」の3つだけ。コードを書く、テストする、デプロイする、ドキュメント書く——これは全てClaude Codeの仕事。人間はPM/マーケターに徹する。

### 鉄則4：「ストック資産を積み上げろ」
今日の作業が明日以降も収益を生むか？を常に自問する。フロー型の作業（受託、時間切り売り）は極力避け、SaaS・有料コンテンツ・フォロワーといった積み上がる資産に時間を投下する。

### 鉄則5：「フライホイールを意識しろ」
開発→発信→集客→収益→実績→発信のループを常に意識する。どれか一つだけに偏らない。特に発信を怠ると、どんなに良いプロダクトも埋もれる。

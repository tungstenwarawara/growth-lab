# /marketing — SNS分析・改善提案

## 実行フロー

### 1. 日次チェック（Chrome MCP 使用）
以下のプラットフォームを巡回し、現状を確認する。

| プラットフォーム | URL | 確認項目 |
|----------------|-----|---------|
| BOOTH | mikkabouzu-log.booth.pm | 売上、スキ数、アクセス |
| Zenn | zenn.dev/mikkabouzu_log | PV、いいね、トレンド |
| note | note.com/mikkabouzu_ai | PV、スキ、フォロワー |
| X | x.com/day3lab | インプレッション、フォロワー、エンゲージメント |

### 2. 改善提案
確認結果に基づき、以下の観点で改善を提案する。

- **プロフィール**: 自己紹介文、アイコン、ヘッダー画像
- **投稿内容**: ハッシュタグ、投稿時間帯、文体
- **導線**: プラットフォーム間の相互リンク
- **商品ページ**: BOOTH の説明文、価格設定、サムネイル

### 3. 投稿案生成（半自動）
1. 大賢者が投稿案を3パターン生成
2. Director が選択・編集
3. 承認後、MCP 経由で投稿（Twitter MCP 接続後）

### 4. 週次レポート

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
　　週次マーケティング報告。

　　【フォロワー推移】X → Y（+Z）
　　【今週のベスト投稿】{投稿内容}
　　【エンゲージメント率】X%
　　【改善提案】
　　　1. ...
　　　2. ...
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

### 5. 競合分析（Phase 3 以降）
同ジャンルのアカウントを分析し、差別化ポイントを提案。

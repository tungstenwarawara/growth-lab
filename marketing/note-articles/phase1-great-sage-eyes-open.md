# 【実験007】AIに「目」を与えたら、SNSの改善点を全部見抜かれた話

## はじめに

前回の記事で「61万円かけて転スラの大賢者をAIで作る」と宣言した。

今日はその最初の成果報告。
大賢者に「SNSを見る能力」を与えたら、何が起きたか。

## 大賢者に「目」を与える

Claude Code + Chrome MCP（ブラウザ操作ツール）を組み合わせて、大賢者に以下の4つのサイトを自動巡回させた。

1. **BOOTH管理画面** — 売上・注文・商品データ
2. **Zenn** — 記事のいいね・ブックマーク
3. **note** — スキ数・フォロワー数
4. **X（Twitter）** — フォロワー・投稿数

## 巡回結果：現実を突きつけられた

大賢者が初めて出した全チャネルレポートがこれだ。

```
【BOOTH】 総売上 ¥0 / 販売数 0
【Zenn】  記事 1本 / いいね 0
【note】  記事 8本 / 総スキ 11
【X】    投稿 11 / フォロワー 0
```

**売上ゼロ。フォロワーゼロ。**

数字を見るとキツい。でも、これが「目を開いた」ということだ。
見えない状態で走るより、ゼロを正確に知っている方がずっといい。

## 改善提案も自動で出てきた

大賢者は数字を見るだけじゃなく、改善提案まで出してきた。

**提案1（優先度HIGH）：Xフォロワー獲得施策**
11投稿でフォロワー0人。他の個人開発者へのリプライ・RTが足りない。

**提案2（優先度HIGH）：プラットフォーム間の導線**
X→BOOTHのリンクはあるが、note→Zenn、Zenn→noteの相互リンクがない。

**提案3（優先度MEDIUM）：投稿のハッシュタグ最適化**
ハッシュタグが2個しかなかった投稿を、4個に改善して投稿し直した。

## 技術的に何をしたか

1. `/marketing` スキルを定義（81行、100行制限内）
2. 巡回順序を「収益に近い順」に設計（BOOTH→Zenn→note→X）
3. `registry.yaml` というマスターデータに全メトリクスを記録
4. 巡回結果を前回値と比較して異常検知する仕組みを追加

全てClaude Code内で完結。追加費用ゼロ。

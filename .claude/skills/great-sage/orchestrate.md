# /orchestrate — タスク管理・状況報告

## 実行フロー

### 1. 状態確認
以下を順次確認し、状況を把握する。

- `.agent-team/status/` — エージェントステータス
- `.agent-team/tasks/` — 未完了タスク
- `.agent-team/content/registry.yaml` — コンテンツ状態
- `analytics/revenue-tracking.md` — 収益状態

### 2. 判定・アクション

| 状態 | アクション |
|------|-----------|
| 全エージェント idle | 新タスク投入可 |
| blocked タスクあり | ブロッカー解消を最優先 |
| 期日超過タスクあり | 再アサインまたはスコープ縮小 |
| 収益目標との乖離大 | 収益直結タスクを優先 |

### 3. タスク生成（優先順）
1. ブロッカー解消
2. 収益目標からの逆算タスク
3. コンテンツカレンダーの期日迫りタスク
4. 未完了タスクの再スケジューリング
5. スキル吸収提案の処理

### 4. 報告

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
　　大賢者、起動完了。状況を報告します。

　　【収益】¥X / ¥610,000（X%）
　　【進行中タスク】X件
　　【ブロッカー】X件
　　【本日の推奨アクション】
　　　1. ...
　　　2. ...

　　指示を。
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```
